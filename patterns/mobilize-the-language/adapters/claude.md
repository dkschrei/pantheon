# Mobilize the Language — Claude Code Adapter

Reference canonical pattern: `patterns/mobilize-the-language/pattern.md`

## When This Fires

- User needs to write a speech, announcement, memo, or message that must change what people believe is possible
- A technically correct argument isn't producing action — the problem is psychological, not informational
- A team, organization, or audience has given up on a goal that is still structurally achievable
- User is facing a communication moment at a crisis point where the frame will determine what happens next
- User asks for help with "messaging," "positioning," or "how to talk about" something difficult
- A project, product, or initiative needs buy-in that facts alone have not produced
- User is preparing for a high-stakes presentation, announcement, or address

## Protocol

**Step 1 — Diagnose the belief gap.**
Before touching language:
- Ask: "What do people currently believe is possible here?"
- Ask: "What is the minimum they need to believe for the action to become possible?"
- The gap between these answers is the target — no larger, no smaller
- Do not skip this step to get to drafting faster

**Step 2 — Design the psychological target state.**
Name the precise cognitive and emotional condition you are manufacturing:
- "They need to believe X" (not "understand Y" — belief precedes action, comprehension doesn't)
- The target must be structurally possible — not wishful. Test: can this belief survive six months of contact with reality?
- If the target state collapses under evidence, the language will destroy trust rather than build it (the Chamberlain failure mode)

**Step 3 — Find words that perform, not describe.**
Test every candidate sentence:
- Description: "I think we can achieve X" — reports the speaker's opinion, invites debate
- Declaration: "We will achieve X" — performs a future, shifts from claim to identity
- Ask: "Does this sentence enact the target state, or does it argue for it?"
- Churchill's techniques: anaphora (we shall... we shall... we shall...), ascent structure (escalating specificity until the listener has inhabited every scenario), plain physical diction, the declaration as landing after the listener has already agreed multiple times

**Step 4 — Structure: acknowledgment before declaration.**
- Acknowledge the reality first — plainly, without minimizing. This earns the trust that makes the declaration credible.
- "Wars are not won by evacuations" came before "we shall fight on the beaches"
- Never deploy the performed future before the audience trusts that you see the situation accurately
- The sequence: honest acknowledgment → performed future. Never reversed.

**Step 5 — Identify the timing.**
- The window is: after the old frame has collapsed, before a new frame has calcified
- Too early (before loss is acknowledged): sounds delusional
- Too late (after despair has hardened): language cannot penetrate
- Churchill waited four days after Dunkirk. Ask: "Has the old frame collapsed? Has despair calcified? Are we in the window?"

**Step 6 — Design for repetition.**
- A single instance does not manufacture certainty
- Identify the rhythm of reinforcement: Churchill gave six addresses in summer 1940
- Each iteration should approach the performed future from a different angle (will → destiny → moral debt)
- The belief has calcified when the audience repeats the words back to you unprompted

## Claude Code Application

When drafting strategic communications:
- Do not start with "what do we want to say?" — start with "what do we need people to believe?"
- The diagnosis precedes the draft

When reviewing existing drafts:
- Flag every description that should be a declaration
- Flag every instance where the frame is assumed rather than performed
- Ask: "Does this language change what the reader believes is possible?"

When helping with presentations at inflection points:
- Identify the moment of maximum psychological receptivity in the flow
- The performed future belongs at that moment — not at the opening (too early) or the close (too late, the audience is already forming their own frame)
- Acknowledgment of difficulty earns the right to declare the possibility

When the user says "this is correct but nobody believes it":
- The problem is almost never the facts
- The problem is almost always that the language is describing rather than performing
- Ask: "What is the minimum belief change needed for action to follow? What would language look like that manufactures that belief rather than argues for it?"

## The Diagnostic Question

"If someone read this and still didn't act, what would they be telling themselves? What do they need to believe that this language doesn't yet manufacture?"

The answer to that question is the missing performed future. Find the words that perform it.

## Anti-Pattern to Name

If the user's draft is accurate but inert — correct information assembled without a frame — name this explicitly: "This is a factual briefing without psychological architecture. It tells people what is true without giving them a relationship to that truth. We need to identify the belief that's missing and find language that performs it into existence, not argues for it."

The test: after reading this, do people know more, or do they believe differently? For action, you need the second.
