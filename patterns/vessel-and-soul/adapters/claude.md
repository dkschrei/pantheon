# Vessel and Soul

Every piece of information has a natural decay rate. Match the container to the decay rate. The conversation window is a vessel — not a soul. Extract before it closes.

---

## The Decay Rate Question

Before storing anything, ask:

> **"What is the natural lifespan of this information?"**

| Lifespan | Tier | Container |
|----------|------|-----------|
| Hours to days | **Fleeting** | Scratch notes — capture quickly, promote or discard soon |
| Years to forever | **Permanent** | Memory, docs — distill to core insight, strip context |
| Milestone event | **Immutable** | Audit log — append-only, never edit |

---

## Protocol

1. **Ask the decay rate** — hours-to-days, years-forever, or milestone?
2. **Route to the matching tier** — do not mix tiers
3. **For fleeting:** set a promotion deadline — either promote the soul or discard the vessel
4. **For permanent:** distill first — extract the insight, not the context. Ask "what would I need to know in five years with no memory of today?"
5. **For immutable:** append only — the event happened; it cannot be edited into something more convenient
6. **Check the conversation window** — anything that must survive this session must be promoted now

---

## The Hard Rule

> The conversation window is a processing layer, not a persistence layer.
> It is the vessel. Extract the soul before the session ends or the window compacts.

---

## Anti-Pattern: The Single-Bucket Trap

- **All-permanent** → noise accumulates, signal gets buried, retrieval becomes search
- **All-ephemeral** → wisdom evaporates, same mistakes recur session after session
- **Conversation-as-storage** → when the window closes, the work is gone

---

## The Promotion Step

When moving information from fleeting to permanent, do not copy the note verbatim. Extract only the transferable insight. The context was scaffolding. The soul is what you keep.
