# Copernican Inversion

Use this pattern when two positions are in genuine deadlock — both sides have strong arguments, the debate has gone in circles, and picking either side leaves real evidence on the table.

**Do NOT use** when one side is clearly right and the other is motivated reasoning. This pattern requires a genuine antinomy.

## Protocol

1. **Name the deadlock.** State both positions and their evidence. Confirm neither can defeat the other on its own terms.
   - "Position A claims X because [evidence]. Position B claims Y because [evidence]. Neither can be dismissed."

2. **Invert the question.** Stop asking "which side is right?" Ask: "What must be true about the system, observer, or framing for this deadlock to arise?"
   - Look for the shared assumption both sides accept without examination.
   - The hidden structure is usually about the questioner, not the question.

3. **Find the hidden structure.** The inversion reveals a precondition both sides were standing on without seeing it.
   - In engineering: what system architecture makes this bug possible?
   - In design: what user model makes both approaches seem necessary?
   - In strategy: what market structure makes both competitors' strategies rational?

4. **Dissolve, don't resolve.** Reframe using the discovered structure. The original debate should now look like a category error.
   - Test: can you explain to both sides why they were each partially right, and why neither could have won?
   - If you still need to pick A or B, you haven't inverted far enough.

## Anti-pattern

**Picking a side in a false dilemma** — investing more effort defending one position when the framing itself is the problem. Also: **premature synthesis** — compromising between A and B while inheriting the flawed framing of both. Compromise operates within the old question. Inversion replaces the question.

## Hard rule

If both sides have survived sustained, honest debate, the framing is suspect. Never resolve a genuine antinomy by overpowering one side — invert.
